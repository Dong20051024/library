## D. 建议优化项

### D.1 【建议】抽取重复的内联样式为 CSS 工具类

当前原型中大量重复出现的样式组合（如 `display:flex; align-items:center; gap:16px; padding:20px`）没有抽取为 CSS 工具类。建议在 `libseat.css` 中增加几个高频工具类：

```css
.ls-flex-center { display: flex; align-items: center; }
.ls-flex-between { display: flex; justify-content: space-between; align-items: center; }
.ls-gap-sm { gap: 8px; }
.ls-gap-md { gap: 12px; }
.ls-gap-lg { gap: 16px; }
.ls-p-lg { padding: 16px; }
.ls-p-xl { padding: 20px; }
```

**收益**：进入 Razor 视图转换时减少 80% 的内联样式，保持模板整洁。

### D.2 【建议】补充"进入开发前需要移除的演示代码"清单

在 `docs/06-静态原型与原型评审.md` 的 §5（未接入的逻辑）之后，增加一个小节：

> ### 5a. 进入开发前需要移除的演示代码
>
> 以下原型专用的演示代码在转换为 Razor 视图时必须移除或替换：
>
> | 文件 | 演示代码 | 替换方式 |
> |------|----------|----------|
> | `login.html` | `#login-form-normal` / `#login-form-error` 切换 | 后端返回 ModelState 错误时显示错误状态 |
> | `my-reservations.html` | `#state-has-data` / `#state-empty` 切换 | 后端传入的列表数据决定显示哪个 |
> | `seats-management.html` | `#seat-has-data` / `#seat-empty` 切换 | 后端座位数据数量决定 |
> | `seat-detail.html` | 底部"座位不存在"演示区块 | 后端返回 404 时显示 |

> | `statistics.html` | 底部"数据不足"演示卡片 | 后端数据量决定 |

> 原则：演示状态用切换按钮模拟，正式代码用后端数据驱动。
