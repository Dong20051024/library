## 2. 本轮任务选择理由

定位 Sprint 2 任务板（docs/项目任务板与迭代记录.md §4.1）后，发现：

| 卡片 | 实际状态 | 本轮选择 |
|------|----------|----------|
| T02-01 ~ T02-05（用户端） | 骨架阶段已包含完整 Controller + View + Service 代码 | **不选择** — 代码已就位，仅 Task Board 状态未更新 |
| **T02-06** 座位管理页列表 | 仅 `ViewBag.Message` 占位桩 | **选择** — 管理端闭环的核心入口页 |
| **T02-07** 新增座位 | 完全未实现 | **选择** — 与 T02-06 同页面弹窗操作 |
| **T02-08** 编辑座位 | 完全未实现 | **选择** — 与 T02-06 同页面弹窗操作 |
| T02-09 删除座位 | 完全未实现 | **附带实现** — 与 T02-06~T02-08 同组，一次完成 |

**选择理由**：T02-06 是"管理端看到座位、点击操作"的入口页，T02-07/T02-08 是最基础的增改操作，三者同属 `/Admin/Seat` 页面，一次实现完整 CRUD 可最大化效率。用户要求"提前解决权限问题"，T02-06~T02-09 涉及管理端操作边界，适合本轮回合完成。

## 8. 默认补足项 / 当前假设

### 默认补足项

| 补足项 | 内容 | 补足理由 |
|--------|------|----------|
| 管理端座位管理使用 Bootstrap Modal 而非单独页面 | 新增/编辑/删除均在同一弹窗内完成，不跳转新页面 | docs/05 PC-08 提到"编辑采用行内编辑或弹窗表单"，弹窗方式与预约管理保持一致的交互模式 |
| 删除座位时使用确认弹窗传递 data-* 属性 | 通过 JS 事件监听从按钮 data 属性读取 Id 和座位号 | 无需单独去后端查询，减少一次 HTTP 请求 |
| 删除时检查有效预约（待签到/使用中） | Service 层用 LINQ `Any()` 检查 Reservation 表 | docs/09 §6.2 E23 已明确此行为 |
| 取消预约增加 owner 校验 | 传递 `studentName` 参数，Service 层比对 `Reservation.StudentName` | docs/09 §9.2 已标注"（假设）学生 A 可以取消学生 B 的预约"并计划修复，本轮提前处理 |

### 当前假设

| 假设 | 影响范围 |
|------|----------|
| 管理端操作不记录操作日志 | 没有审计记录 |
| 不分页 | 列表一次性加载所有数据 |
| 密码明文存储 | 课堂项目，LocalDB 本地使用 |
