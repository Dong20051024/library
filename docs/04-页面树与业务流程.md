## 2. 用户端页面树

### 2.1 页面树结构

```
                        ┌──────────────┐
                        │   用户首页    │  ← P1 系统入口
                        │  /Home/Index │
                        └──────┬───────┘
                               │
              ┌────────────────┼────────────────┐
              ▼                ▼                 ▼
     ┌────────────────┐  ┌──────────┐  ┌────────────────┐
     │   座位列表页    │  │  体验账号  │  │   我的预约页    │
     │  /Seat/List    │  │  切换组件  │  │ /Reservation/Mine
     └────────┬───────┘  │ (全局导航) │  └────────┬───────┘
              │           └──────────┘           │
              ▼                                  │
     ┌────────────────┐                          │
     │   座位详情页    │  ← 点击单个座位进入       │
     │ /Seat/Detail/  │                          │
     │        {id}    │                          │
     └────────┬───────┘                          │
              │                                  │
              ▼                                  │
     ┌────────────────┐                          │
     │   预约提交页    │  ← 点击"预约"按钮         │
     │ /Reservation/  │                          │
     │ Create?seatId= │                          │
     └────────┬───────┘                          │
              │ 提交成功后自动跳转                  │
              └──────────────────────────────────┘
```

### 2.2 页面详细定义

| # | 页面名称 | 路由 | 优先级 | 页面类型 | 说明 |
|---|----------|------|--------|----------|------|
| P1 | 用户首页 | `/` 或 `/Home/Index` | P1 | 入口页 | 系统名称、欢迎语、导航入口；全局导航栏包含体验账号切换组件 |
| P2 | 座位列表页 | `/Seat/List` | P0 | 列表页 | 座位网格/表格展示，三种状态用颜色区分，支持按楼层筛选 |
| P3 | 座位详情页 | `/Seat/Detail/{id}` | P1 | 详情页 | 展示单个座位的完整信息，含"预约"按钮（仅空闲状态显示） |
| P4 | 预约提交页 | `/Reservation/Create?seatId={id}` | P0 | 表单页 | 选日期 + 选时段 + 确认提交 |
| P5 | 我的预约页 | `/Reservation/Mine` | P1 | 列表页 | 当前体验账号的所有预约记录，含取消操作入口 |

### 2.3 页面跳转规则

| 起点 | 触发动作 | 终点 | 条件 |
|------|----------|------|------|
| 用户首页 | 点击"查看座位" | 座位列表页 | 无条件 |
| 用户首页 | 点击"我的预约" | 我的预约页 | 无条件 |
| 用户首页 | 切换体验账号 | 用户首页（刷新） | 下拉选择账号 |
| 任何用户端页面 | 切换体验账号 | 用户首页 | 切换后重定向到首页 |
| 座位列表页 | 点击空闲座位卡片 | 座位详情页 | 座位状态=空闲 |
| 座位列表页 | 点击已预约/使用中座位 | 无跳转 | 不可点击 |
| 座位详情页 | 点击"预约"按钮 | 预约提交页 | 座位状态=空闲 |
| 座位详情页 | 点击"返回列表" | 座位列表页 | 无条件 |
| 预约提交页 | 提交成功 | 我的预约页 | 预约创建成功 |
| 预约提交页 | 提交失败 | 预约提交页（停留） | 显示错误信息 |
| 我的预约页 | 点击"取消预约" | 我的预约页（刷新） | 确认弹窗→确定→取消成功 |
| 我的预约页 | 点击"返回首页" | 用户首页 | 无条件 |

## 4. 用户主链路

### 4.1 预约座位链路（核心链路，必须闭环）

```
 [用户首页] ──→ [座位列表页] ──→ [座位详情页] ──→ [预约提交页] ──→ [我的预约页]
    ①             ②               ③                ④                ⑤
```

**详细步骤：**

| 步骤 | 页面 | 用户操作 | 系统响应 |
|------|------|----------|----------|
| ① | 用户首页 | 切换到自己的体验账号，点击"查看座位" | 进入座位列表页 |
| ② | 座位列表页 | 浏览座位，看到绿色（空闲）座位；可下拉筛选楼层 | 显示所有座位，不同状态不同颜色 |
| ③ | 座位详情页 | 点击一个空闲座位的卡片进入详情；确认设施标签（有插座、靠窗等） | 显示座位号、楼层、区域、设施、当前状态 |
| ④ | 预约提交页 | 点击"预约"按钮；选择日期和时段（如 14:00-17:00）；点击"确认预约" | 系统校验时段是否可用，创建预约记录 |
| ⑤ | 我的预约页 | 提交成功后自动跳转至此，看到刚提交的记录状态为"待签到" | 预约状态=待签到，对应座位变为已预约 |

**闭环确认**：链路走到这里，用户端核心流程已完成。后续签到和释放由管理员在管理端完成。

### 4.2 取消预约链路

```
 [我的预约页] ──→ 点击"取消预约" ──→ 弹出确认框 ──→ 确认取消 ──→ 列表刷新
     ①                                                          ②
```

| 步骤 | 页面 | 用户操作 | 系统响应 |
|------|------|----------|----------|
| ① | 我的预约页 | 找到"待签到"状态的预约，点击"取消预约" | 弹出确认对话框 |
| ② | 我的预约页 | 在确认框点击"确定" | 预约状态变为"已取消"，对应座位恢复"空闲"；列表刷新 |

### 4.3 体验账号切换链路（辅助链路）

```
 [任意用户端页面] ──→ 点击顶部导航栏的账号名 ──→ 下拉选择其他账号 ──→ 跳转回首页
      ①                          ②                          ③
```

| 步骤 | 页面 | 用户操作 | 系统响应 |
|------|------|----------|----------|
| ① | 任意用户端页面 | 看到导航栏显示当前体验账号名（如"学生1"） | 维护当前会话中的用户身份 |
| ② | 任意用户端页面 | 点击账号名，下拉列表中选择"学生2" | 更新会话中的用户身份 |
| ③ | 用户首页 | 自动跳转回首页 | 所有数据基于新账号身份展示 |

## 5. 管理主链路

### 5.1 签到与释放链路（核心管理链路，必须闭环）

```
 [管理员登录页] ──→ [预约管理页] ──→ 确认签到（待签到→使用中）
      ①                ②                ③
                                           ──→ 释放座位（使用中→已完成/空闲）
                                                  ④
```

| 步骤 | 页面 | 操作 | 系统响应 |
|------|------|------|----------|
| ① | 管理员登录页 | 输入账号密码（预设：admin / 123456） | 校验通过，写入 Session，跳转到预约管理页 |
| ② | 预约管理页 | 查看所有预约列表，可按状态筛选（如筛选"待签到"） | 显示预约记录：学生名、座位号、时段、状态 |
| ③ | 预约管理页 | 找到对应预约，点击"确认签到" | 预约状态从"待签到"→"使用中"，座位从"已预约"→"使用中" |
| ④ | 预约管理页 | 学生离馆后，点击"释放座位" | 预约状态从"使用中"→"已完成"，座位从"使用中"→"空闲" |

**闭环确认**：到这里，一次完整的座位生命期（空闲→已预约→使用中→空闲）走完。

### 5.2 座位管理链路（辅助管理链路）

```
 [座位管理页] ──→ 查看座位列表 ──→ 新增 / 编辑 / 删除座位
     ①                             ②
```

| 步骤 | 页面 | 操作 | 系统响应 |
|------|------|------|----------|
| ① | 座位管理页 | 查看当前所有座位，可以看到每个座位的楼层、区域、编号、设施、状态 | 分页列表展示 |
| ② | 座位管理页 | 新增：填写楼层、区域、座位号、设施标签 → 保存 | 新座位立即出现在用户端座位列表 |
|  |  | 编辑：修改座位信息 → 保存 | 修改内容立即生效 |
|  |  | 删除：确认删除（如有预约记录则禁止删除） | 删除成功或提示无法删除 |

## 6. 状态流转

### 6.1 座位状态四态流转

```
     ┌──────────────────────────────────────────────────┐
     │                  座位状态流转图                     │
     │                                                  │
     │    (学生提交预约)          (管理员确认签到)          │
     │  ┌──────────┐          ┌──────────┐              │
     │  │  空闲     │ ──────→  │  已预约   │ ──────→      │
     │  │ (绿色)    │          │ (黄色)    │              │
     │  └────┬─────┘          └──────────┘              │
     │       ↑                                            │
     │       │ (取消预约)                                  │
     │       │                                            │
     │       │         (管理员释放座位)                     │
     │       └─────────────────┐                          │
     │                         │                          │
     │                    ┌────┴─────┐                    │
     │                    │  使用中   │ ←────────────────  │
     │                    │ (红色)    │                    │
     │                    └──────────┘                    │
     └──────────────────────────────────────────────────┘

     * 注意：座位没有"已完成"或"已取消"状态
     * "已完成"和"已取消"是预约的状态，不是座位的
     * 座位只有：空闲、已预约、使用中
```

### 6.2 预约状态四态流转

```
     ┌──────────────────────────────────────────────────┐
     │                 预约状态流转图                      │
     │                                                  │
     │    (学生提交预约)          (管理员确认签到)          │
     │  ┌──────────┐          ┌──────────┐              │
     │  │  待签到   │ ──────→  │  使用中   │ ──────→      │
     │  └────┬─────┘          └──────────┘              │
     │       │                                            │
     │       │ (学生取消)                                  │
     │       ▼                                            │
     │  ┌──────────┐          ┌──────────┐               │
     │  │  已取消   │          │  已完成   │ ←──────────── │
     │  └──────────┘          └──────────┘               │
     │                              ↑                     │
     │                         (管理员释放座位)             │
     └──────────────────────────────────────────────────┘
```

### 6.3 座位 ↔ 预约状态联动表

| 座位状态 | 对应的预约状态 | 说明 |
|----------|---------------|------|
| 空闲 | （无关联预约） | 可被预约 |
| 已预约 | 待签到 | 学生已预约但尚未到馆签到 |
| 使用中 | 使用中 | 学生已签到，正在使用座位 |
| 空闲 | 已完成 | 使用结束，座位已释放（预约记录保留） |
| 空闲 | 已取消 | 学生取消预约，座位释放（预约记录保留） |

### 6.4 不允许的操作

| 操作 | 原因 |
|------|------|
| 学生取消状态为"使用中"的预约 | 已签到入座后不可线上取消，需管理员介入处理 |
| 同一座位同一时段重复预约 | 时段冲突校验阻止 |
| 删除仍有预约记录的座位 | 外键约束阻止（或业务层拦截） |
| 已取消/已完成的预约再次修改状态 | 生命周期已结束 |

## 9. 暂不扩展页面

> 以下页面已评估但刻意排除在本期范围外。不是"想不到"，而是"想好了不做"。所有"暂不扩展"页面的功能已通过已有页面或其他方式覆盖。

| 页面 | 不做的理由 | 已有替代方案 |
|------|-----------|-------------|
| 注册/登录页（用户端） | 用体验账号切换替代 | 全局导航栏的体验账号切换组件 |
| 座位地图/平面图页 | 开发成本高，列表模式够用 | 座位列表页的网格展示 + 颜色区分 |
| 预约历史归档页 | 刚上线数据不多，我的预约页可以查看近期记录 | 我的预约页 |
| 个人中心/个人信息页 | 没有真实账号体系，没有个人信息需要维护 | 体验账号切换组件可满足 |
| 公告/通知页 | 偏离核心预约流程 | 用户首页可展示简短公告 |
| 操作日志页 | 课堂项目不需要审计追溯 | 无替代（且不需要） |
| 管理员账号管理页 | 本期只有一个管理员 | 种子数据预置即可 |
| 黑名单/信用分页 | 运营功能，初始不需要 | 无替代（且不需要） |
| 帮助/常见问题页 | 系统足够简单，不需要帮助页面 | 无替代（且不需要） |
| 数据导出页 | 刚上线数据量小，截图即可汇报 | 统计页可满足查看需求 |
