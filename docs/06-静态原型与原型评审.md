## 2. 采用的视觉方向

### 主方向：校园服务感

整体视觉实现严格遵循 `docs/05-页面卡与UI规范.md` 中的设计系统：

| 规范维度 | 实现情况 |
|----------|----------|
| 主色 `#2B5F8A`（学院蓝） | 用户端导航栏、主按钮、管理端侧边栏选中态 |
| 主色浅 `#E8F0F8` | 表格表头、卡片选中背景、提示区域 |
| 主色深 `#1A3D5C` | 管理端侧边栏背景、按钮 hover |
| 辅助色 `#D4A843`（书卷金） | 图表高亮柱、头像背景 |
| 成功 `#27AE60` / 警告 `#E67E22` / 危险 `#D35400` | 座位状态条、状态标签、按钮 |
| 禁用 `#95A5A6` | 已完成/已取消标签、禁用文字 |
| 圆角 8px / 12px | 按钮 8px、卡片 12px |
| 卡片阴影 `0 2px 8px rgba(0,0,0,0.06)` | 所有卡片统一样式 |
| 字体系统 | 系统无衬线字体栈，H1=24px/700，正文=16px/400 |

### 用户端 vs 管理端

| 维度 | 用户端 | 管理端 |
|------|--------|--------|
| 布局策略 | 移动端优先（手机竖屏框架 `ls-phone-frame`） | 桌面端优先（左侧固定侧边栏 + 主内容区） |
| 导航 | 顶部蓝色导航栏 56px，含体验账号切换 | 深蓝侧边栏 220px + 白色顶部栏 60px |
| 内容呈现 | 卡片式布局，大间距，引导性强 | 表格布局，紧凑高效，筛选器+操作区 |
| 手机适配 | 手机竖屏框架 + 响应式网格（<576px 单列） | 小屏侧边栏折叠为水平菜单 |

### 自定义样式文件位置

```
prototype/static-v1/assets/css/libseat.css
```

所有非 Bootstrap 默认的样式均集中在此文件中，包括：

- 全部 CSS 变量（设计令牌）
- 用户端导航（`.ls-topnav`）
- 座位卡片网格（`.ls-seat-grid`, `.ls-seat-card`）
- 自定义按钮系统（`.ls-btn`, `.ls-btn-primary`, `.ls-btn-danger` 等）
- 状态标签（`.ls-badge-*`）
- 管理端侧边栏（`.ls-sidebar`）
- 管理端布局系统（`.ls-admin-wrapper`, `.ls-admin-main`）
- 空状态（`.ls-empty-state`）
- Toast 通知（`.ls-toast`）
- 模态弹窗（`.ls-modal`）
- 统计页柱状图（纯 CSS）
- 手机端框架（`.ls-phone-frame`）
- 响应式断点覆盖

## 4. 已实现的状态类型

要求至少展示成功状态、空状态、异常提示中的两类，本原型覆盖如下：

### 成功状态

| 页面 | 状态 | 演示方式 |
|------|------|----------|
| `user/reservation-create.html` | ✅ 预约提交成功 | 点击"确认预约"按钮 → 绿色 Toast "预约提交成功！正在跳转…" → 2秒后自动跳转到我的预约页 |
| `user/my-reservations.html` | ✅ 取消预约成功 | 点击"取消预约" → 弹窗确认 → 绿色 Toast "预约已取消" |
| `admin/reservations.html` | ✅ 签到/释放成功 | 点击"签到"或"释放座位"按钮 → 绿色 Toast "操作成功" |

### 空状态

| 页面 | 状态 | 演示方式 |
|------|------|----------|
| `user/my-reservations.html` | 📭 无预约记录 | 顶部演示切换按钮 → 点击"空状态" → 显示空状态插图和"去选座"按钮 |

### 异常/错误状态

| 页面 | 状态 | 演示方式 |
|------|------|----------|
| `admin/login.html` | ⚠️ 密码错误 | 顶部演示切换按钮 → 点击"错误状态" → 表单显示红色边框 + 错误提示"账号或密码错误" |

## 5. 未接入的逻辑

以下功能在原型中通过手动点击或静态展示模拟，未实现真实业务逻辑。正式开发时必须接入：

| 功能 | 当前处理方式 | 正式开发要求 |
|------|-------------|-------------|
| 体验账号切换 | 点击下拉列出 5 个账号，切换后无实际数据变化 | 切换后所有页面数据随账号变化 |
| 预约提交 | 点击按钮显示成功 Toast 并跳转 | 后端校验时段冲突后创建 Reservation 记录 |
| 取消预约 | 弹窗确认后显示 Toast | 后端更新 Reservation.Status + Seat.Status |
| 签到确认 | 点击按钮显示 Toast | 后端事务更新预约+座位状态 |
| 释放座位 | 点击按钮显示 Toast | 后端事务更新预约+座位状态 |
| 筛选按钮 | 前端切换 active 样式 | 向后端发送筛选参数重新查询 |
| 楼层筛选 | 前端按钮切换 | 向后端发送 floor 参数过滤座位 |
| 新增/编辑座位 | 弹窗表单，点击保存关闭弹窗 | 后端写入/更新 Seat 表 |
| 删除座位 | 弹窗确认后关闭 | 后端校验关联预约记录后删除 |
| 登录验证 | 点击直接跳转预约管理页 | 后端校验 Username + Password |
| 统计数据 | 硬编码 7 天数据 | 后端 GROUP BY 统计后返回 |
| 分页 | 静态显示页码 | 后端分页返回数据 |

## 6. 用户端核心页面手机布局说明

所有用户端页面采用手机竖屏优先布局，包含以下特征：

- **顶部状态栏 + 导航**：固定顶部 56px 蓝色导航栏，左侧系统名，右侧体验账号
- **手机框架**：576px 以上显示带圆角和阴影的手机壳效果（`.ls-phone-frame`）
- **单列纵向流**：内容从上到下排列，按钮全宽，适合拇指操作
- **底部安全留白**：`.ls-page` 底部 `padding-bottom: 80px`
- **按钮尺寸**：全宽按钮高度 48px，适合手指点击
- **表单**：输入框高度 48px（移动端），间距 16px

### 电脑端查看提示

在 992px 以上宽屏上，用户端页面会自动居中并限制最大宽度 960px；在 576px 以上还会套上手机外壳边框效果。建议用浏览器开发者工具切换到移动端视图查看最佳效果。

## 7. 管理端核心页面桌面布局说明

所有管理端页面采用桌面端优先布局：

- **侧边栏**：固定左侧 220px 深蓝色侧边栏，含菜单项和选中态指示
- **顶部栏**：60px 白色顶部栏，显示当前页面标题 + 管理员信息 + 退出按钮
- **主内容区**：右侧自适应内容区，灰色背景
- **表格**：隔行变色、列宽固定、操作按钮靠右
- **筛选器**：列表上方按钮组 + 搜索框

在 767px 以下，侧边栏折叠为水平菜单，顶部栏高度缩减。
