## 1. 本轮选择理由

### 1.1 Sprint 1 当前状态

Sprint 0 打造的工程骨架中，Service 层、Controller 层和 ViewModel 层已包含完整业务逻辑（含事务、冲突校验、Session 读写、异常处理）。所有 10 张功能性任务卡（T01-01 ~ T01-10）的代码均已在骨架阶段就位。

本轮以此为基础，选择 **3 张紧邻的、覆盖用户端主链路的任务卡** 进行验收和最终精调：

| 卡片 | 标题 | 选择理由 |
|------|------|----------|
| **T01-10** | 用户端布局（导航栏 + 账号切换） | 用户端所有页面的基础骨架，必须先就位 |
| **T01-07** | 座位列表页 | P0 核心页面，用户链路第一步 |
| **T01-08** | 预约提交页 GET/POST | P0 核心页面，包含写库操作 |

### 1.2 递进关系

```
T01-10 (布局) → T01-07 (选座) → T01-08 (预约提交) → 【写库验证】
                                                      ↓
                                           本轮完成线：至少一次
                                           真实写库 + 查库
```

此 3 张卡片构成一个**可验收闭环**：用户能打开网站 → 看到导航栏和当前账号 → 浏览座位列表 → 进入预约提交页 → 完成预约 → 在我的预约页看到结果。

### 3.2 管理端侧边栏高亮

`Areas/Admin/Views/Shared/_AdminLayout.cshtml` 中增加了一段内联 JavaScript，根据当前 URL 自动给匹配的侧边栏 `<a>` 添加 `active` 类：

```javascript
var path = window.location.pathname.toLowerCase();
document.querySelectorAll('.admin-sidebar .nav-link').forEach(function(link) {
    if (link.getAttribute('href') && path.indexOf(link.getAttribute('href').toLowerCase()) >= 0) {
        link.classList.add('active');
    }
});
```
