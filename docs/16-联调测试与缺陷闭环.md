## 3. 缺陷发现与修复：Bug #001

### 3.1 问题描述

**现象**：管理员登录 POST 请求返回 `AmbiguousMatchException`，错误信息提示 `LoginController.Index` 和 `LoginController.Logout` 同时匹配请求。

**触发测试**：`admin-flow.spec.ts` → 管理员登录 → 预约管理页（@smoke）
**首次发现**：Playwright 烟雾测试运行

### 3.2 根因分析

Admin 区域控制器同时使用了**属性路由**和**常规路由**两套机制：

```
// 控制器级别属性路由（冲突源）
[Route("Admin/[controller]")]
public class LoginController : Controller { ... }

// Program.cs 中同时存在的常规路由
app.MapAreaControllerRoute("Admin", "Admin", "Admin/{controller=Login}/{action=Index}/{id?}");
```

当 `[Route("Admin/[controller]")]` 作用于控制器类且 `[HttpPost]` 无显式 `[Route]` 模板时，**所有 POST 动作均继承类级别路由** `Admin/Login`。因此：

- `LoginController.Index(POST)` → 属性路由 `Admin/Login` + POST
- `LoginController.Logout(POST)` → 属性路由继承 `Admin/Login` + POST
- 两个端点注册到同一 URL+HTTP 方法 → AmbiguousMatchException

### 3.3 修复方案

**方案**：移除所有 Admin 控制器的 `[Route("Admin/[controller]")]` 属性路由，完全依赖 `MapAreaControllerRoute` 常规路由。

| 文件 | 修改 |
|------|------|
| `Controllers/Admin/LoginController.cs` | 删除 `[Route("Admin/[controller]")]` |
| `Controllers/Admin/ReservationController.cs` | 删除 `[Route("Admin/[controller]")]` |
| `Controllers/Admin/SeatController.cs` | 删除 `[Route("Admin/[controller]")]` |
| `Controllers/Admin/StatisticsController.cs` | 删除 `[Route("Admin/[controller]")]` |

**验证**：`dotnet build` 0 error 0 warning → `npx playwright test --grep @smoke` 7/7 通过（修复前 6/7）。

### 3.4 经验教训

- **不要混用属性路由和常规路由**。同一控制器区域应选择一种路由机制，避免 ASP.NET Core 路由歧义。
- `[Route("controller")]` 在控制器类级别时，所有动作继承同一模板，容易意外创建重复端点。
- Playwright 烟雾测试在首次运行即捕获此问题，表明自动化测试对路由配置变更的敏感性极高。
