## 2. 项目背景

很多高校图书馆都存在"座位被占但人不在"的现象。学生到图书馆找不到空座，或者看到了空座但不敢坐，因为不知道座位上的人是不是只是暂时离开。

目前学校没有统一的座位管理系统，靠的是"书包占座"这种原始方式，效率低、纠纷多。一个轻量的在线预约系统就能解决这个问题——学生提前预约座位，到馆签到入座，离开时释放座位，管理员也能在后台统一管理。

本项目模拟"入职后独立负责第一个小项目"的场景，从立项到交付全流程走一遍，重点锻炼全栈开发能力和项目落地思维。

## 3. 目标用户

| 用户角色 | 说明 |
|----------|------|
| 普通学生（读者） | 使用图书馆座位预约功能，查座位、约座位、取消预约 |
| 图书馆管理员 | 管理座位状态、查看预约记录、查看统计数据 |

本期不做注册流程，用户端使用体验账号切换的方式模拟不同用户；管理端用轻量 Session/Cookie 登录。

---

## 4. 项目目标

1. 实现一个可运行的图书馆座位预约 Web 系统，核心流程走通。
2. 学生能在线查看座位、提交预约、取消预约。
3. 管理员能管理座位和预约数据，查看基础统计。
4. 代码提交到 Git 仓库，结构清晰，能作为课程作业或作品集项目。

## 7. 页面规模与页面分布

本期计划开发 **8 个页面**，分为用户端和管理端两个区域：

### 用户端（5 页）

| 页面 | 路由 | 说明 |
|------|------|------|
| 首页 | `/` 或 `/Home/Index` | 系统入口，展示欢迎信息和快速入口 |
| 座位列表 | `/Seat/List` | 展示所有座位及其状态，支持按状态/楼层筛选 |
| 座位详情 | `/Seat/Detail/{id}` | 查看单个座位信息，点击"预约"跳转预约页 |
| 预约提交 | `/Reservation/Create?seatId=xx` | 选择时段、确认预约 |
| 我的预约 | `/Reservation/Mine` | 查看个人预约记录，可执行取消操作 |

### 管理端（3 页）

| 页面 | 路由 | 说明 |
|------|------|------|
| 管理员登录 | `/Admin/Login` | 固定账号密码 + Session/Cookie 登录 |
| 预约管理 | `/Admin/Reservation` | 查看全部预约，支持状态筛选和搜索 |
| 座位管理 | `/Admin/Seat` | 座位的增删改查 |
| 数据统计 | `/Admin/Statistics` | 预约趋势图、座位利用率统计 |

> 管理端采用独立 Area 或独立 Controller 前缀，与用户端路由隔离。4 个功能共用同一套后台布局，算作"管理端"一个整体模块，页面数量按 3 页计算（登录页 + 管理主页/功能页）。

## 8. 关键业务主流程

### 8.1 预约座位流程（核心流程）

```
学生打开首页 → 查看座位列表 → 选择空闲座位 →
填写预约时段（例如 14:00-17:00）→ 提交预约 →
系统创建预约记录（状态：待签到）→
学生到馆后由管理员标记"已入座"（模拟签到）→
使用结束后管理员标记"已释放" → 座位恢复空闲
```

> 说明：本期"签到"操作用管理员后台手动确认来模拟，不做扫码或自助签到机。

### 8.2 取消预约流程

```
学生在"我的预约"中找到待签到的记录 → 点击取消 →
系统将预约状态改为"已取消" → 座位恢复空闲
```

### 8.3 管理员管理座位流程

```
管理员登录后台 → 进入座位管理 → 新增/编辑/删除座位 →
每个座位包含：楼层、区域、座位号、设施标签（插座/靠窗/电脑位等）
```

## 10. 风险与假设

### 风险

| 风险 | 可能后果 | 应对措施 |
|------|----------|----------|
| 项目时间被课程作业挤占 | 做不完核心功能，最后仓促交差 | 先做核心预约流程，再做管理端，统计页最后搞；时间不够就砍统计页 |
| 页面数量控制不住 | 一个一个写页面写到崩溃 | 先定死 8 页，做完不动；Bootstrap 样板代码复用，不要每个页面重新发明样式 |
| SQL Server LocalDB 安装/兼容问题 | 本地跑不起来，越调越烦 | 课前确认 LocalDB 能用；实在不行和老师确认能否换成 SQLite |
| AI 生成代码不稳定，改错越改越乱 | 花大量时间 debug，挫败感强 | 每次让 AI 改代码前先 git commit，改坏了直接回滚；不要一次性让 AI 改大段代码 |
| 第一次用 EF Core，数据库迁移卡住 | 数据库表结构对不上，程序启动报错 | 严格按官方模板走 Add-Migration / Update-Database，不要手动改数据库 |

### 假设

- 开发环境为 Windows 11 + Visual Studio 2022 + .NET 8 SDK
- SQL Server LocalDB 随 Visual Studio 安装附带可用
- 不考虑并发冲突（两个学生同时约同一个座位的情况直接报错提示）
- 演示时使用浏览器直接操作，不需要部署到服务器
- 最终通过 GitHub/Gitee 仓库提交，不要求部署上线
