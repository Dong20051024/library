### 2.3 Service 职责

#### ② ISeatService.GetSeatList

```csharp
// SeatService.cs — 返回 SeatListResult DTO（不含 UI 状态字段）
public SeatListResult GetSeatList(string? floor)
{
    var query = _context.Seats
        .Where(s => s.IsActive);

    if (!string.IsNullOrEmpty(floor))
        query = query.Where(s => s.Floor == floor);

    var seats = query
        .OrderBy(s => s.Floor)
        .ThenBy(s => s.Area)
        .ThenBy(s => s.SeatNumber)
        .Select(s => new SeatListItem
        {
            Id = s.Id,
            SeatNumber = s.SeatNumber,
            Floor = s.Floor,
            Area = s.Area,
            Status = s.Status,           // 返回 enum，CSS class 由 Controller 层映射
            Facilities = s.Facilities
        })
        .ToList();

    return new SeatListResult
    {
        Seats = seats,
        CurrentFloor = floor,
        Floors = _context.Seats
            .Where(s => s.IsActive)
            .Select(s => s.Floor)
            .Distinct()
            .OrderBy(f => f)
            .ToList()
    };
}
```

| 职责 | 说明 |
|------|------|
| 查询激活的座位 | `WHERE IsActive = true` |
| 筛选楼层 | `WHERE Floor = @floor`（可选参数） |
| 提供楼层列表 | `SELECT DISTINCT Floor FROM Seat WHERE IsActive`，供筛选下拉框使用 |
| 返回 DTO 而非 ViewModel | 返回 `SeatListResult`（不含 `ErrorMessage`、`StatusCssClass` 等 UI 状态），Controller 层组装最终 ViewModel |

#### ④ IReservationService.CreateReservation（核心方法）

```csharp
public void CreateReservation(int seatId, string studentName, DateTime startTime, DateTime endTime)
{
    // 校验 1：时间段合理性
    if (startTime >= endTime)
        throw new BusinessException("结束时间必须晚于开始时间");
    if (startTime < DateTime.Now)
        throw new BusinessException("预约开始时间不能早于当前时间");

    using var transaction = _context.Database.BeginTransaction();
    try
    {
        // 校验 2：座位存在且空闲
        var seat = _context.Seats.FirstOrDefault(s => s.Id == seatId)
            ?? throw new BusinessException("所选座位不存在或已被删除");
        if (!seat.IsActive)
            throw new BusinessException("该座位已禁用，无法预约");
        if (seat.Status != SeatStatus.空闲)
            throw new BusinessException("该座位当前状态无法预约");

        // 校验 3：时段重叠（核心防重复校验）
        var conflictCount = _context.Reservations
            .Count(r => r.SeatId == seatId
                     && r.StartTime < endTime
                     && r.EndTime > startTime
                     && (r.Status == ReservationStatus.待签到
                      || r.Status == ReservationStatus.使用中));
        if (conflictCount > 0)
            throw new BusinessException("该座位在您选择的时间段已被预约，请选择其他时段或座位");

        // 写入预约记录
        var reservation = new Reservation
        {
            SeatId = seatId,
            StudentName = studentName,
            StartTime = startTime,
            EndTime = endTime,
            Status = ReservationStatus.待签到,
            CreatedAt = DateTime.Now
        };
        _context.Reservations.Add(reservation);

        // 更新座位状态
        seat.Status = SeatStatus.已预约;

        _context.SaveChanges();
        transaction.Commit();
    }
    catch
    {
        transaction.Rollback();
        throw;
    }
}
```

| 职责 | 说明 |
|------|------|
| 时间段合理性校验 | EndTime > StartTime；StartTime > 当前时间 |
| 座位存在性校验 | Seat.Id 有效 |
| 座位可用性校验 | IsActive=true 且 Status=空闲 |
| 时段重叠校验 | **核心防重逻辑**，见下方详细说明 |
| 写入 Reservation | INSERT 一条新记录 |
| 更新 Seat.Status | 空闲 → 已预约 |
| 事务提交 | 两个写入在同一个事务中 |

#### 时段重叠校验算法

```
两个时间区间 [A开始, A结束] 和 [B开始, B结束] 重叠的条件：
      A开始 < B结束  AND  A结束 > B开始

在 SQL 中的实现：
    WHERE SeatId = @seatId
      AND StartTime < @endTime
      AND EndTime > @startTime
      AND Status IN (0, 1)    -- 只检查"待签到"和"使用中"
```

### 2.4 DataAccess 动作

| 步骤 | DataAccess 操作 | 影响的表 | SQL 模式 |
|------|----------------|----------|----------|
| ② 列表查询 | 读取 | Seat | `SELECT ... FROM Seat WHERE IsActive=1 [AND Floor=@floor] ORDER BY ...` |
| ③ 详情查询 | 读取 | Seat | `SELECT ... FROM Seat WHERE Id=@id` |
| ④ 时段冲突检查 | 读取 | Reservation | `SELECT COUNT(*) FROM Reservation WHERE SeatId=@sid AND StartTime<@end AND EndTime>@start AND Status IN (0,1)` |
| ④ 创建预约 | 写入 | Reservation | `INSERT INTO Reservation (...) VALUES (...)` |
| ④ 更新座位 | 写入 | Seat | `UPDATE Seat SET Status=1 WHERE Id=@sid` |
| ⑤ 我的预约 | 读取 | Reservation | `SELECT ... FROM Reservation WHERE StudentName=@name ORDER BY CreatedAt DESC` |

### 2.5 数据库变化

| 操作前数据库状态 | 操作 | 操作后数据库状态 |
|-----------------|------|-----------------|
| Seat: Status=0(空闲) | ④ 提交预约 | Seat: Status=1(已预约) |
| Reservation: (无此记录) | ④ 提交预约 | Reservation: INSERT 新行, Status=0(待签到) |
| 其他数据不变 | 不涉及 | — |

**事务边界**：④ 的两个写入（INSERT Reservation + UPDATE Seat）在同一事务中，任一失败整体回滚。

### 2.6 状态流转（链路一）

```
预约提交前                             预约提交后
──────────                            ──────────
Seat.Status = 空闲 (0)    ─────→     Seat.Status = 已预约 (1)
                                      Reservation.Status = 待签到 (0)
                                          (新记录)
```

### 2.7 异常与边界情况（链路一）

| 编号 | 场景 | 触发条件 | 系统行为 | 错误提示 |
|------|------|----------|----------|----------|
| E01 | 座位不存在 | `seatId` 无效 | 返回 NotFound 视图 | "座位信息不存在，请返回列表重新选择" |
| E02 | 座位已禁用 | `IsActive = false` | POST 时 BusinessException，停留在表单页 | "该座位已禁用，无法预约" |
| E03 | 座位已被占用 | `Status != 空闲` | GET 时直接提示；POST 时 BusinessException | "该座位当前无法预约" / "该座位当前状态无法预约" |
| E04 | 时段冲突 | 同一座位同一时段已有有效预约 | BusinessException | "该座位在您选择的时间段已被预约，请选择其他时段或座位" |
| E05 | 结束早于开始 | `EndTime <= StartTime` | ModelState 验证失败（前端+后端） | "结束时间不能早于开始时间" |
| E06 | 开始时间在过去 | `StartTime < DateTime.Now` | BusinessException | "预约开始时间不能早于当前时间" |
| E07 | 并发提交（极端情况） | 两个请求同时通过冲突检查，同时写入 | 事务提交时第二个会成功但座位状态已变（不做乐观锁） | (本期不处理高并发，容忍极小概率下两条记录写入) |
| E08 | 数据库连接失败 | SQL Server 不可用 | catch 异常，返回错误提示 | "提交失败，请检查网络后重试" |

### 3.3 Service 职责

```csharp
public void CancelReservation(int reservationId)
{
    using var transaction = _context.Database.BeginTransaction();
    try
    {
        var reservation = _context.Reservations
            .Include(r => r.Seat)
            .FirstOrDefault(r => r.Id == reservationId)
            ?? throw new BusinessException("预约记录不存在");

        // 只有"待签到"状态允许取消
        if (reservation.Status != ReservationStatus.待签到)
            throw new BusinessException("当前预约状态不允许取消（仅待签到状态可以取消）");

        // 更新预约状态
        reservation.Status = ReservationStatus.已取消;

        // 恢复座位为空闲
        reservation.Seat.Status = SeatStatus.空闲;

        _context.SaveChanges();
        transaction.Commit();
    }
    catch
    {
        transaction.Rollback();
        throw;
    }
}
```

| Seat: Status=1(已预约) | 取消预约 | Seat: Status=0(空闲) |

```csharp
// 返回 AdminReservationResult DTO（不含 UI 状态字段）
public AdminReservationResult GetAllReservations(ReservationStatus? statusFilter)
{
    var query = _context.Reservations
        .Include(r => r.Seat)
        .AsQueryable();

    if (statusFilter.HasValue)
        query = query.Where(r => r.Status == statusFilter.Value);

    var reservations = query
        .OrderByDescending(r => r.CreatedAt)
        .Select(r => new AdminReservationItem
        {
            Id = r.Id,
            SeatNumber = r.Seat.SeatNumber,
            Floor = r.Seat.Floor,
            Area = r.Seat.Area,
            StudentName = r.StudentName,
            StartTime = r.StartTime,
            EndTime = r.EndTime,
            Status = r.Status             // 返回 enum，CSS class / 按钮可见性由 Controller 映射
        })
        .ToList();

    return new AdminReservationResult
    {
        Reservations = reservations,
        CurrentStatus = statusFilter
        // StatusOptions、StatusCssClass 等 UI 字段在 Controller 层组装
    };
}
```

| 职责 | 说明 |
|------|------|
| 查询预约 + JOIN 座位 | `INNER JOIN Seat ON Reservation.SeatId = Seat.Id` |
| 按状态筛选 | `WHERE Status = @status`（可选） |

#### IReservationService.CheckIn

```csharp
public void CheckIn(int reservationId)
{
    using var transaction = _context.Database.BeginTransaction();
    try
    {
        var reservation = _context.Reservations
            .Include(r => r.Seat)
            .FirstOrDefault(r => r.Id == reservationId)
            ?? throw new BusinessException("预约记录不存在");

        // 状态校验：只有"待签到"可以签到
        if (reservation.Status != ReservationStatus.待签到)
            throw new BusinessException("当前预约状态不允许签到（仅待签到状态可签到）");

        // 更新预约：待签到 → 使用中
        reservation.Status = ReservationStatus.使用中;

        // 更新座位：已预约 → 使用中
        reservation.Seat.Status = SeatStatus.使用中;

        _context.SaveChanges();
        transaction.Commit();
    }
    catch
    {
        transaction.Rollback();
        throw;
    }
}
```

#### IReservationService.Release

```csharp
public void Release(int reservationId)
{
    using var transaction = _context.Database.BeginTransaction();
    try
    {
        var reservation = _context.Reservations
            .Include(r => r.Seat)
            .FirstOrDefault(r => r.Id == reservationId)
            ?? throw new BusinessException("预约记录不存在");

        // 状态校验：只有"使用中"可以释放
        if (reservation.Status != ReservationStatus.使用中)
            throw new BusinessException("当前预约状态不允许释放（仅使用中状态可释放）");

        // 更新预约：使用中 → 已完成
        reservation.Status = ReservationStatus.已完成;

        // 更新座位：使用中 → 空闲
        reservation.Seat.Status = SeatStatus.空闲;

        _context.SaveChanges();
        transaction.Commit();
    }
    catch
    {
        transaction.Rollback();
        throw;
    }
}
```

### 5.4 DataAccess 动作

| 步骤 | DataAccess 操作 | 影响的表 | SQL 模式 |
|------|----------------|----------|----------|
| ① 登录查询 | 读取 | Admin | `SELECT * FROM Admin WHERE Username=@u AND Password=@p` |
| ② 预约列表 | 读取 | Reservation + Seat | `SELECT r.*, s.Floor, s.Area, s.SeatNumber FROM Reservation r INNER JOIN Seat s ON r.SeatId=s.Id [WHERE r.Status=@status] ORDER BY r.CreatedAt DESC` |
| ③ 签到 | 写入 | Reservation, Seat | `UPDATE Reservation SET Status=1 WHERE Id=@id` + `UPDATE Seat SET Status=2 WHERE Id=@sid` |
| ④ 释放 | 写入 | Reservation, Seat | `UPDATE Reservation SET Status=2 WHERE Id=@id` + `UPDATE Seat SET Status=0 WHERE Id=@sid` |

### 5.5 数据库变化

| 操作前 | 操作 | 操作后 |
|--------|------|--------|
| Reservation: Status=0(待签到), Seat: Status=1(已预约) | ③ 确认签到 | Reservation: Status=1(使用中), Seat: Status=2(使用中) |
| Reservation: Status=1(使用中), Seat: Status=2(使用中) | ④ 释放座位 | Reservation: Status=2(已完成), Seat: Status=0(空闲) |

| 体验账号不能自定义（固定 5 个） | 切换账号时不校验密码，不做登录态检查 | 改为真实用户注册/登录，需新建 User 表 + 密码验证 |
| 不校验"取消预约的操作人是否是预约本人" | 学生 A 登录后可以取消学生 B 的预约 | 加 `WHERE StudentName = @currentUser` 条件 |
| 管理端操作不记录操作日志 | 没有"谁在什么时间做了什么"的审计记录 | 新建 ReservationLog 表，每次操作 INSERT 一条日志 |
| 不处理高并发预约 | 两条同时提交的预约请求可能都通过时段检查 | 使用数据库 `UPDLOCK` 或应用层分布式锁 |
| 不分页 | 列表一次性加载所有数据 | 引入分页参数 `page` 和 `pageSize`，Service 层用 `Skip/Take` |
| 管理端不强制要求 Session 超时时间 | 默认浏览器 Session 过期（约 20 分钟无操作） | 在 Program.cs 中显式配置 `IdleTimeout` |
