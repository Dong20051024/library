# 图书馆座位预约系统

轻量化的图书馆座位预约 Web 系统。学生在线查看座位状态、预约座位、取消预约；图书馆管理员后台管理座位和预约数据，确认签到和释放座位。

> **最终版本**：v1.0-final | **技术栈**：ASP.NET Core MVC (.NET 8) + Razor + SQL Server LocalDB + EF Core + Bootstrap 5

## 项目简介

一个面向校园场景的图书馆座位预约系统，支持：

- **学生端**：查看空闲座位、按楼层筛选、查看座位详情、预约座位、查看我的预约、取消预约（仅限待签到状态）
- **管理端**：管理员登录/登出、查看所有预约及按状态筛选、确认签到、释放座位、座位增删改管理、数据统计

**核心业务链路**：

```
查看座位 → 预约座位 → 管理员签到 → 使用中 → 管理员释放 → 座位空闲
```

**体验机制**：学生使用体验账号（无需注册），导航栏下拉即可切换 5 个预设学生账号，每个账号有不同的预约历史，方便演示不同场景。

## 运行步骤

### 环境要求

| 依赖 | 说明 |
|------|------|
| Windows 10/11 | LocalDB 仅在 Windows 上可用 |
| .NET 8 SDK | [下载](https://dotnet.microsoft.com/download/dotnet/8.0) |
| SQL Server LocalDB | 随 VS2022 安装，或[单独安装](https://learn.microsoft.com/sql/database-engine/configure-windows/sql-server-express-localdb) |
| Visual Studio 2022（推荐） | 或 VS Code + C# Dev Kit |

### 快速开始

```bash
# 1. 进入项目目录
cd src/LibrarySeatReservation.Web

# 2. 还原依赖
dotnet restore

# 3. 创建数据库（执行 EF Core 迁移，种子数据自动插入）
dotnet ef database update

# 4. 运行项目
dotnet run

# 5. 浏览器访问
# 用户端：http://localhost:5000/
# 管理端：http://localhost:5000/Admin/Login
```

> 实际端口以 `Properties/launchSettings.json` 中 `applicationUrl` 为准。默认同时监听 `https://localhost:62259` 和 `http://localhost:62260`。

详细数据库初始化说明见 [database/README.md](./database/README.md)。

## 数据库初始化

### 方式：EF Core Code First

本项目使用 **Code First** 方式。实体类（`Models/Entities/`）和 Fluent API 配置（`Data/`）定义表结构，通过迁移文件同步到 SQL Server LocalDB。

### 首次建库

```bash
dotnet ef database update
```

### 3 张数据表

| 表名 | 说明 |
|------|------|
| `Seats` | 座位（楼层、区域、座位号、状态、设施、是否激活） |
| `Reservations` | 预约记录（学生名、座位 FK、起止时间、状态） |
| `Admins` | 管理员（用户名、密码、显示名） |

完整字段定义见 `docs/08-数据库设计.md`，初始化说明见 `database/README.md`。

### 迁移文件

已存在于 `src/LibrarySeatReservation.Web/Migrations/`。首次运行 `dotnet ef database update` 时自动应用。

## 种子数据说明

种子数据在首次启动时通过 `Program.cs` 调用 `SeedData.Initialize()` 自动插入。

**14 个座位**：3 楼 A 区 6 个 + B 区 4 个，5 楼 C 区 4 个。覆盖空闲、已预约、使用中三种状态。

**6 条预约记录**：覆盖全部 4 种预约状态（待签到/使用中/已完成/已取消），分配给 5 个不同学生账号。

**1 个管理员账号**：admin / 123456

**幂等性**：首次运行后检测 `Seats` 表已有记录则自动跳过，重复启动不重复插入。
